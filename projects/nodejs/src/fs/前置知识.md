# Node.js fs 模块前置核心知识汇总

## 一、同步 vs 异步（最核心）
### 1. 同步（Sync）
- **特点**：阻塞主线程，代码等待执行完成才继续
- **表现**：服务器会卡住，无法处理其他请求
- **API**：`fs.readFileSync`、`fs.writeFileSync`
- **使用场景**：仅用于**程序启动初始化**，绝不用于接口/服务中

### 2. 异步（Async）
- **特点**：非阻塞，不影响主线程
- **API 风格**
  - 回调函数：`fs.readFile(file, callback)`
  - Promise：`fs.promises.readFile` / `fs.readFile` + `util.promisify`
- **使用场景**：99% 业务逻辑必须用异步
- **Node.js 核心优势**：非阻塞 I/O

---

## 二、路径处理（跨平台必学）
### 1. 相对路径
- 格式：`./file.txt`
- 风险：依赖**运行命令的目录**，极易找不到文件

### 2. 绝对路径
- 格式：完整路径
- 推荐写法：`path.join(__dirname, 'file.txt')`
- 作用：无论在哪运行，都能精准找到文件

### 3. 路径分隔符问题
- Windows：`\`
- Linux/Mac：`/`
- **解决方案**：必须使用 `path` 模块拼接，不要手动拼字符串

### 4. path 模块 5 个核心属性
- `root`：根路径 `/`
- `dir`：文件所在目录
- `base`：文件名 + 后缀 `file.txt`
- `ext`：扩展名 `.txt`
- `name`：文件名（无后缀）`file`

## 三、字符编码与 Buffer
### 1. 字符编码
- 读取文本文件**必须指定编码**
- 默认：`utf8`
- 不指定 → 返回 Buffer（二进制）

### 2. Buffer
- Node.js 处理**二进制数据**的对象
- 用于：图片、视频、音频、压缩包等非文本文件
- 理解：二进制 = 0 和 1 的缓冲区

---

## 四、文件流（Stream）处理大文件
### 1. 普通方式（readFile/writeFile）
- 一次性把文件**全部读进内存**
- 风险：大文件会导致**内存溢出、程序崩溃**
- 适用：小文件（几 KB ~ 几 MB）

### 2. 流式处理（createReadStream / createWriteStream）
- 特点：**分片读取**，一次只读一小块
- 内存占用极低、稳定
- 适用：
  - 大文件（10MB+）
  - 视频、日志、压缩包
- 理解：像水管输水，不是把整个水库搬回家

---

## 五、文件描述符 & 系统权限
### 1. 文件描述符（fd）
- 操作系统给打开文件的**数字编号**
- 作用：复用文件句柄，提升性能
- 场景：高频写入日志、高性能操作

### 2. 文件权限
- Node.js 受系统用户权限限制
- 常见错误：`EACCES`（权限不足）
- 原因：写入系统目录、只读文件、无权访问文件夹

---

## 六、fs 模块使用黄金准则
1. **永远优先异步**，拒绝线上使用 Sync
2. **永远用 path 模块拼接路径**，不手写字符串
3. **读取文本必须指定编码 utf8**
4. **大文件必须用 Stream**
5. **路径必须用绝对路径（__dirname）**
6. **错误必须捕获**（try/catch 或回调 err 判断）

---

## 七、核心概念速查表
| 概念 | 关键点 | 最佳实践 |
|------|--------|----------|
| 同步/异步 | 是否阻塞主线程 | 只用异步 |
| 路径 | 跨平台兼容 | path.join + __dirname |
| 编码 | 不乱码 | 指定 utf8 |
| Buffer | 二进制文件 | 图片/视频专用 |
| Stream | 大文件 | 10MB+ 必须用流 |
| 权限 | 系统限制 | 处理 EACCES 错误 |
